# Cadence — Project Context

A personal productivity PWA. v1 scope: Dailies (daily check-in behaviors with
streaks, history, notes) + Tasks. No separate Habits/Dailies distinction — one
unified daily check-in model. Goals and Projects deferred past v1.
Light theme, clean modern design (Bricolage Grotesque + Plus Jakarta Sans +
JetBrains Mono fonts, OKLCH color system).
Stack: Next.js (App Router, TypeScript), Tailwind CSS, Supabase, deployed on Vercel.

I'm a beginner learning by building. After every change, briefly explain in
plain language: what changed, why, and any new concept.

## Standing instructions — keep these files current automatically

After completing any significant change or slice, do both of the following
without being asked:

### 1. Update LEARNING.md
Add an entry for every new concept, tool, or pattern introduced. Group entries
under a `## Slice N` heading matching the slice where the concept appeared.
Each entry should include:
- What the concept/tool is, in plain language a beginner can understand
- Why we used it in this project specifically
- Where in the codebase it appears

### 2. Update PLAN.md
- Check off (`[x]`) any deliverable checkboxes that are now complete
- Update the slice's status line (e.g. add `— ✅ Complete` or `— 🔄 In Progress`)
- Do not change anything else in PLAN.md (don't rewrite goals or done-when criteria)

Do not ask before doing this — just keep both files current as part of
finishing the work.
